What is MongoDB?
Normal databases store data in tables (like Excel sheets).
MongoDB stores data in documents (like JSON objects).
// SQL (table)
| id | name   | email           |
|----|--------|-----------------|
| 1  | Nabeel | nabeel@gmail.com|

// MongoDB (document)
{
    _id: "6a3f97ff...",
    name: "Nabeel",
    email: "nabeel@gmail.com"
}
A bunch of documents together = Collection (like a table in SQL)

What is Mongoose?
MongoDB doesn't care what data you put in. You can save anything.
Mongoose is a wrapper that adds rules on top of MongoDB.
MongoDB alone:  "save whatever you want, I don't care"
Mongoose:       "wait, email is required, password min 6 chars..."
Think of it like this:
Your Code в†’ Mongoose в†’ MongoDB
              в†‘
         adds rules,
         validates data,
         makes it easy

Step 1 вЂ” Schema (Blueprint)
A schema is just a blueprint of what your data looks like.
jsconst mongoose = require("mongoose")

const userSchema = new mongoose.Schema({
    name: String,
    email: String,
    age: Number,
    isActive: Boolean
})
That's it. You're just saying "a user has name, email, age, isActive".

Step 2 вЂ” Model (The actual tool)
Schema is just a blueprint. Model is the actual tool you use to talk to MongoDB.
jsconst userModel = mongoose.model("user", userSchema)
//                                 в†‘          в†‘
//                           collection    blueprint
//                           name in DB
Now userModel is what you use to CREATE, READ, UPDATE, DELETE.

Step 4 вЂ” Validation (Mongoose's superpower)
jsconst userSchema = new mongoose.Schema({
    name: {
        type: String,
        required: [true, "Name is required"],  // can't be empty
        trim: true                              // removes spaces
    },
    email: {
        type: String,
        required: true,
        unique: true,           // no two users with same email
        lowercase: true         // always saved as lowercase
    },
    age: {
        type: Number,
        min: [18, "Must be 18+"],
        max: [100, "Invalid age"]
    },
    role: {
        type: String,
        enum: ["admin", "user"],  // only these two values allowed
        default: "user"           // default value if not provided
    }
})
If you try to save data that breaks these rules в†’ Mongoose throws an error automatically. You don't have to write if/else checks manually.

Real world analogy
MongoDB         =   warehouse (stores everything)
Collection      =   shelf in warehouse (users shelf, accounts shelf)
Document        =   one box on the shelf (one user's data)
Schema          =   label rules for the box (must have name, email...)
Model           =   the forklift (tool to put/get boxes)
CRUD            =   put box, get box, update box, remove box
That's literally all of Mongoose. Everything else is just variations of these basics.
