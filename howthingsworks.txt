1. Atlas vs Compass

Many beginners get confused here.

MongoDB Atlas

Think of Atlas as:

Google Drive for databases

It stores your database on MongoDB's cloud servers.

Your data lives here:

Internet
  ↓
MongoDB Atlas

So even if your laptop shuts down, data remains.

MongoDB Compass

Think of Compass as:

File Explorer for MongoDB

It does NOT store data.

It only lets you view data.

Like:

Atlas = Actual Hard Disk
Compass = File Explorer
What we did

We connected Compass to Atlas.

Compass
    ↓
Atlas Cluster

Now Compass can show:

backend-ledger
 └── users

Laptop

MongoDB equivalent:

For you:

JWT

This confuses almost everyone at first.

Let's use your college.

Imagine after paying fees, college gives you:

ID Card

Next day when you enter:

Security doesn't ask:

Pay fees again

They check:

Show ID card

If valid:

Come inside

JWT is exactly this ID card.

Without JWT

User logs in.

Every request:

Enter email
Enter password

Again and again.

Very annoying.

With JWT

Login once.

Server creates:

jwt.sign(...)

Produces:

abc.xyz.123

This is the JWT token.

Server sends:

{
  "token":"abc.xyz.123"
}
Then What?

Frontend stores token.

Usually:

Cookie
or
Local Storage
Next Request

User visits profile.

Frontend sends:

GET /profile

Authorization: Bearer abc.xyz.123

Server checks token.

If valid:

Yes, this is Nabeel.

No need for password again.

Real Life Analogy

Password:

College Fee Receipt

JWT:

Student ID Card

You use fee receipt once.

You use ID card daily.

What is inside JWT?

When you do:

jwt.sign(
{
   userId:user._id
},
process.env.JWT_SECRET
)

Server creates:

{
   "userId":"123"
}

and locks it with a secret key.

Why JWT_SECRET?

Suppose hacker creates fake token:

{
   "userId":"admin"
}

Without secret:

Anyone can become admin.

JWT_SECRET prevents this.

Only your server knows:

JWT_SECRET=abcdef123

So only your server can create valid tokens.
