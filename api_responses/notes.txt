📒 Note: Insufficient balance check
/api/transactions/ (normal transfer) → always checks balance before allowing transfer, for anyone (regardless of user role)
/api/transactions/system/initial-funds → separate endpoint, no balance check at all — it's meant to inject new money into the system, like a bank creating funds
Rule: it's not "system user bypasses the check" — it's "system user has access to a different endpoint that was never built with a check"
